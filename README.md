The Mission
This repository serves as the official documentation for my ITAI1378 Lab Assignment. The goal was simple: master the workflow, get the tools under control, and establish a solid baseline for version control and interactive computing. No fluff, just the work.

What this repo will entail:
It tracks the foundational moves required for modern engineering documentation:

Version Control: Setting up the GitHub ecosystem, cloning the repo, and handling the commit-and-push lifecycle via GitHub Desktop.

Interactive Environments: Wrangling Jupyter Notebooks—executing cells, handling Markdown, and getting the logic running.

The Contents
My_First_Notebook.ipynb: The primary artifact. It’s a clean demonstration of basic Python execution and Markdown structuring. It’s what happens when you turn code into a reality.

How to run:
When you need to inspect the mechanics:

Pull the repo: Clone this down to your machine.

Access: Throw the .ipynb file into Google Colab or your local Jupyter environment.

Execute: Run the cells. It’s optimized to work straight out of the box.
